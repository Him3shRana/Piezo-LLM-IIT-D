Bugs found and fixed this session:

    --model ... was being passed literally (a copy-paste placeholder left in a command) causing FileNotFoundError — fixed by using --model medium or a real path.
    LAMMPS command hardcodes KOKKOS flags (-k on g 1 -sf kk -pk kokkos ...) — failed when the resolved lmp binary lacked KOKKOS, or when run without sourcing env.sh / without a GPU. Resolved by using the correct binary + sourcing env.sh on a GPU compute node.
    lammps_common.py's build_nvt_input() and build_npt_input() used restart {restart_every_steps} {output_restart}.* — the .* wildcard made LAMMPS write a new numbered restart file every checkpoint (e.g. nvt_equilibration.restart.100, .200, ...). Fixed by removing .* so it overwrites a single restart file instead. Applied via:

bash

   sed -i 's/f"restart         {restart_every_steps} {output_restart}\.\*"/f"restart         {restart_every_steps} {output_restart}"/g' lammps_common.py

This fix only affects future stage runs — already-completed stages keep their old numbered checkpoint files, which were manually cleaned up with rm *.restart.[0-9]*.
